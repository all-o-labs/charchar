# Teach charchar

Scan your project to understand its existing character design conventions, then generate a `.charchar` configuration file that all other charchar skills will reference for consistency.

Run this once when you start using charchar on an existing project.

## Workflow

### Step 1: Scan Project Assets

Search the project for character-related files:

```
Scan targets:
├── **/*.svg          (vector characters)
├── **/*.png / *.jpg  (raster sprites, sheets)
├── **/character*     (character-related directories)
├── **/sprite*        (sprite directories)
├── **/avatar*        (avatar directories)
├── **/*.css          (character style definitions)
├── **/*.ts / *.js    (character component code)
└── **/*.json         (sprite metadata, animation data)
```

### Step 2: Analyze Existing Patterns

Extract design patterns from found assets:

**Visual Style:**
- Outline style (solid, tapered, none)
- Shading method (cel, gradient, flat)
- Line weight range
- Color saturation range

**Proportions:**
- Head-to-body ratio
- Common character sizes (pixel dimensions)
- Aspect ratios

**Color Palette:**
- Extract dominant colors across all characters
- Identify the palette style (limited, full-color)
- Detect common hue ranges

**Technical Patterns:**
- File format conventions (SVG, PNG, Canvas)
- Naming conventions
- Directory structure
- Animation format (CSS, sprite sheet, JS)

### Step 3: Detect Conventions

Analyze code for established patterns:

```javascript
// Look for existing style systems
const conventions = {
  cssVariables: findCSSCustomProperties('--char-*', '--sprite-*'),
  components: findComponents('*character*', '*sprite*', '*avatar*'),
  constants: findConstants('HEAD_RATIO', 'SPRITE_SIZE', 'PALETTE'),
  assetPaths: findAssetDirectories(),
};
```

### Step 4: Generate .charchar Config

Create a `.charchar` file at the project root:

```yaml
# .charchar — Project Character Design Configuration
# Generated by /teach-charchar
# All charchar skills reference this file for project consistency

project:
  name: "My Game"
  engine: "phaser"        # detected game engine/framework
  asset_dir: "src/assets/characters"

style:
  outline: solid          # solid | tapered | none
  shading: cel            # cel | gradient | flat
  line_weight:
    outline: 3
    internal: 1.5
    detail: 1
  detail_density: medium  # minimal | low | medium | high

proportions:
  head_ratio: 3           # body heights per head
  base_size: 32x32        # standard sprite size
  eye_style: large_round

palette:
  mode: limited           # limited | full
  max_colors_per_char: 16
  primary_hues: [220, 35, 350]  # detected dominant hues
  saturation_range: [40, 70]
  skin_tones:
    - "#f4c89a"
    - "#d4a070"
    - "#a06830"

animation:
  format: spritesheet     # spritesheet | css | canvas | skeletal
  fps: 12
  standard_anims: [idle, walk, run, attack, hurt, death]

conventions:
  naming: kebab-case      # kebab-case | camelCase | snake_case
  file_format: svg        # svg | png | canvas
  component_pattern: "Character{Name}"
  css_prefix: "--char-"

existing_characters:
  - name: hero
    path: src/assets/characters/hero.svg
    colors: ["#3b6bb5", "#d4a520", "#f4c89a"]
  - name: villain
    path: src/assets/characters/villain.svg
    colors: ["#6b2d7b", "#1a1a2e", "#c83030"]
```

### Step 5: Verify and Report

Output a summary of what was found and configured:

```
charchar project scan complete
═══════════════════════════════

Found:
  3 character files (SVG)
  1 sprite sheet (PNG + JSON)
  12 CSS custom properties (--char-*)
  2 character components

Detected Style:
  Outline: solid (3px)
  Shading: cel-shading
  Proportions: 3-head ratio
  Palette: limited (16 colors)

Generated: .charchar

All charchar skills will now use these settings
for consistent character design in this project.
```

## Do

- Run this before using other charchar skills on an existing project
- Review the generated `.charchar` file and adjust if the auto-detection missed nuances
- Commit `.charchar` to version control so the whole team shares the same config
- Re-run after major style changes to update the configuration

## Don't

- Skip this step and let skills use generic defaults on an established project
- Edit `.charchar` without understanding what each field controls
- Run on an empty project (there's nothing to scan — just start designing)
- Ignore the scan results if they reveal inconsistencies in your existing assets
